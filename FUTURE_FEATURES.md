# 今後実装予定の機能

## SNS自動告知・フライヤー自動生成機能

### 概要

政党管理者が演説予定をSNS（X・Instagram・Threads・LINE）へ自動投稿できる機能。毎朝8:00に確定済みの演説予約情報をもとにフライヤー画像と投稿テキストを自動生成し、各SNSへ配信する。

---

### 実装予定の内容

#### 1. SNSアカウント登録（政党管理者メニュー）

政党管理者メニューに以下のSNSアカウント情報を登録・保存できるフォームを追加する。

- X（旧Twitter）アカウント名・APIキー
- Instagramアカウント名・アクセストークン
- Threadsアカウント名・アクセストークン
- LINE公式アカウントID・チャンネルアクセストークン

アカウント情報はSupabaseの `settings` テーブルに政党ごとに暗号化して保存する。

#### 2. フライヤー自動生成

確定済みの演説予約情報をもとに、以下の内容を含むフライヤー画像（PNG形式）をCanvas APIで自動生成する。

- 政党名・候補者名
- 演説日・時間帯・場所名
- 地図上の場所（ピンのスクリーンショット等）
- 配信URLがある場合はQRコード

フライヤーは**演説日ごとに1枚**生成する。デザイン雛形は政党ごとにカスタマイズ可能とする（ロゴ画像・テーマカラーを設定できる形）。

#### 3. 自動投稿スケジューリング

毎朝8:00に当日・翌日以降の確定済み演説情報を取得し、自動投稿を実行する。投稿内容は以下の通り。

- フライヤー画像（演説日ごと1枚）
- 本文テキスト（演説場所・日時・候補者名の一覧）
- 配信URLがある場合はリンクを含める

---

### 技術的な前提条件

現在のシステム構成（静的HTML + Supabase）では、定時実行とSNS APIへのサーバーサイド認証が必要なため、以下のインフラ追加が必要になる。

#### 必要なインフラ

**Supabase Edge Functions**
毎朝8:00のcronジョブとして動作させる。各SNS APIへのリクエストをサーバーサイドで実行し、アクセストークンをセキュアに管理する。

```
supabase/functions/
  daily-sns-post/
    index.ts   # メイン処理（cron: 毎日8:00 JST）
```

もしくは **GitHub Actions** を使ったcronワークフローでも対応可能。

#### 各SNS APIの対応状況

| SNS | API名 | 画像投稿 | 自動投稿 | 備考 |
|---|---|---|---|---|
| X | X API v2 | ✅ | ✅ | Basic以上のプランが必要 |
| Instagram | Graph API | ✅ | ✅ | Meta Business Suite経由 |
| Threads | Threads API | ✅ | ✅ | 2024年公開・Metaアカウント必要 |
| LINE | Messaging API | ✅ | ✅ | LINE公式アカウント必要 |

---

### 実装スケジュールの目安

この機能は以下のフェーズが完了してから実装を開始する。

1. **予約・抽選・管理機能の安定化**（現在進行中）
2. **大人数によるテスト実施**（ステージング環境での検証）
3. **必要に応じた別ページ分割・PWA化**
4. **本機能の実装開始**

予約機能が確実に固まった後に実装することで、告知した演説情報と実際の予約状態の不整合リスクを防ぐ。

---

### 段階①：手動投稿補助機能（暫定対応）

バックエンド追加前の暫定対応として、以下の機能を先行実装することも検討する。

- 政党管理者メニューの「今日の投稿を作成」ボタンでフライヤー画像をブラウザ上で生成・ダウンロード
- 投稿テキストをクリップボードにコピー
- 手動でSNSに貼り付けて投稿する運用

これにより、フライヤー作成の手間はゼロにしつつ、バックエンドなしで運用を開始できる。
